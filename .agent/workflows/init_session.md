# Session Initialization Workflow

Before starting work on any new feature, bug fix, or project modification, follow these steps:

1. **Review Project Context**
   - Search the project for relevant documentation (e.g., `README.md`, `project_knowledge.md`).
   - Use the agent to summarize the current project state so you have up-to-date context on the architecture and workflow rules. 


2. **Update the README.md**
   - Update the README.md with the summarize of the current project from the README.md, project_knowledge.md before making any modifications. Keep the README.ME updated.
   

3. **Verify Dev Environment**
   - Run the development start workflow to ensure the project runs properly before you touch it.
   - Use `/dev_start` or run `npm run dev`.
